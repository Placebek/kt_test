# Тренажёр · Алгоритмы и Базы данных

Локальный тренажёр для подготовки к вступительному комплексному тестированию
в магистратуру. Девять вариантов по предметам «Алгоритмы» и «Базы данных»
(в вариантах 5 и 9 ещё и «Английский язык», в варианте 8 — «Программирование»/
«Информатика», в варианте 9 — ещё и «Тест на готовность к обучению») с полными
объяснениями к каждому ответу — **почему он правильный и почему остальные неверны**.
Плюс варианты 10–16 — методические тесты «Информатика (методика оқыту)» и новый
предмет «ИТ-тест» (сети, БД, Excel, Python, логика), собранные из отдельной папки
источников `questions/infor-programming/` (см. отдельный раздел ниже).

## Как открыть
    
Просто откройте файл **`index.html`** двойным щелчком в любом браузере
(Chrome, Edge, Firefox). Никакой установки, сервера или интернета не требуется —
всё работает офлайн из файла (`file://`). Данные тестов лежат в обычных `.js`-файлах,
а прогресс сохраняется только в вашем браузере (localStorage).

8 вопросов в алгоритмах и 4 вопроса в базах данных — **трассировка**: дан
короткий фрагмент кода на C++ (для алгоритмов) или мини-таблица с SQL-запросом
(для баз данных), и нужно определить, что он выведет/вернёт, а не просто
вспомнить определение. Темы: асимптотика, рекурсия, деревья, графы
(BFS/DFS/Дейкстра), динамическое программирование, хеширование, кучи,
амортизированный анализ, техника двух указателей и скользящего окна — в
алгоритмах; ACID, ключи, JOIN, нормализация (2НФ/3НФ), индексы, уровни
изоляции транзакций, представления, NoSQL, ограничения SQL, блокировки,
триггеры и ER-модель — в базах данных. Объяснения — на русском, как в
вариантах 1–9.

### Как считается балл
- **Одиночный выбор** (алгоритмы, программирование, английский): 1 балл за верный
  вариант, иначе 0.
- **Множественный выбор** (базы данных, информатика): до 2 баллов за вопрос. `2` —
  выбранное множество точно совпадает с верным; `1` — верных выбрано не меньше, чем
  неверных (и хотя бы один верный); иначе `0`.

## Возможности

- **📝 Сдать экзамен** — конструктор смешанного экзамена: отмечаете нужные предметы
  (например «Алгоритмы» + «Базы данных» + «Английский язык»), и для **каждого**
  предмета отдельно и случайно выбирается один из имеющихся вариантов — независимо
  от того, какие варианты выпали остальным предметам (алгоритмы могут прийти из
  варианта 3, базы — из варианта 9, английский — из варианта 5, и т.д.). Все вопросы
  собираются в один экзамен с таймером и разбором в конце.
- **Два режима:** «Практика» (мгновенная проверка и объяснение после каждого ответа)
  и «Экзамен» (таймер, разбор в конце).
- **Панель-навигатор** по вопросам с цветами «отвечен / верно / неверно / текущий».
- **Работа над ошибками** — прорешать заново только неверные вопросы.
- **Практика по темам** (фильтр по теме) и поиск по вопросам.
- **Автосохранение** прогресса и история результатов по каждому варианту.
- **Тёмная тема** (кнопка 🌙) и **горячие клавиши** (A–H — выбор, →/Enter — далее,
  ← — назад, F — флажок).
- **Показ вашего исходного ответа** из реальной попытки для сравнения.

## Про правильные ответы (важно)

Ключ ответов выведен **не** по синим галочкам на скриншотах (это ваш выбор во время
попытки, часть из них неверна), а по таблице официальных результатов из папки
`results/` + экспертной проверке. Каждый вопрос помечен уровнем достоверности ключа:
- `official` — ответ подтверждён официальным баллом;
- `derived` — правильный набор восстановлен экспертно (там, где вы ошиблись);
- значок **«⚠ спорный ответ»** — официальный ключ академически сомнителен; в объяснении
  указано и то, что засчитывает система, и как правильно на самом деле
  (например, вариант-2 / алгоритмы, вопросы про рекурсию Фибоначчи и циклы).

## Объяснения от ИИ (по желанию)

Кроме предзаписанных объяснений (работают офлайн и бесплатно) есть кнопка
**«Спросить ИИ»** для более глубокого разбора. Она использует ваш собственный ключ
провайдера (OpenRouter), который вводится один раз и хранится только в вашем браузере
(`localStorage`). Это дополнение, а не обязательная часть: основной разбор всегда
доступен без интернета.

> Живой запрос к ИИ может блокироваться при открытии через `file://` (ограничения CORS
> у некоторых провайдеров). Если ИИ не отвечает — откройте сайт через локальный сервер
> (например `python -m http.server` в папке проекта) или пользуйтесь предзаписанными
> объяснениями.

## Структура проекта

```
index.html            — точка входа (тесты)
flashcards.html       — отдельная страница карточек для запоминания
css/styles.css        — стили и тёмная тема
css/flashcards.css    — стили страницы карточек (использует переменные styles.css)
js/app.js             — экраны, состояние, навигация (тесты)
js/flashcards.js      — логика страницы карточек
js/scoring.js         — подсчёт баллов (обе схемы, по полю multi)
js/storage.js         — сохранение прогресса (localStorage), общий для тестов и карточек
js/math.js            — офлайн-рендер формул (\frac, \sqrt, ^, _ и т.п.)
js/ai.js              — опциональный запрос к ИИ (свой ключ)
data/v*-*.js           — по 2–3 файла на вариант: вопросы, ключи, объяснения
data/flashcards-*.js   — по файлу на предмет: карточки {subject, topic, front, back}
results/              — исходные таблицы официальных результатов
questions/            — исходные скриншоты вопросов
```
